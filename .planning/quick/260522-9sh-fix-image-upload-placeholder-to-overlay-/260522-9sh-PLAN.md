<objective>
Replace the in-editor "Uploading image…" placeholder and inline error insertion with non-editable DOM overlays appended to the `.tiptap-editor` wrapper element.

Purpose: Bloggers are confused when "Uploading image…" and error text appear as editable content inside the ProseMirror rich-text area. Moving feedback outside ProseMirror makes the editor state clean and the UX unambiguous.
Output: Updated controller with overlay helpers; updated CSS with overlay rules; old `.tiptap-image-placeholder` rule removed.
</objective>

    - On success: call `this._removeUploadOverlay()` before `this.editor.chain().focus().setImage(...)`.

    Rewrite `showInlineError(message)`:
    - Remove all `this.editor.commands.insertContent(...)` and `querySelectorAll` calls.
    - Create a `div` with class `tiptap-error-banner`. Set `contenteditable` to `"false"`. Set `textContent` to `message`.
    - Append to `this.element`.
