## Objective

Two targeted bug fixes in the admin blog section:
1. "View" link missing on non-published blogs in index
2. Slug auto-generation broken when form submits empty string

## Bug Analysis

**Bug 1 — View link condition too strict:**
`app/views/admin/blogs/index.html.erb` line 75 condition:
`blog.slug.present? && blog.published_at && blog.published_at <= Time.current`
Only shows View for past-published blogs. Drafts and future-dated posts never get a link.

**Bug 2 — Slug ||= falsy on empty string:**
`app/models/blog.rb` `generate_slug`:
`self.slug ||= title.parameterize if title.present?`
HTML form submits `""` for a blank slug field. `"" || value` = `""` in Ruby (empty string is truthy), so the auto-generation never fires.

## Tasks

### Task 1: Fix View link condition + slug blank? guard

**Files:** `app/views/admin/blogs/index.html.erb`, `app/models/blog.rb`

**Actions:**

1. In `app/views/admin/blogs/index.html.erb` line 75, change:
   ```erb
   <% if blog.slug.present? && blog.published_at && blog.published_at <= Time.current %>
   ```
   to:
   ```erb
   <% if blog.slug.present? %>
   ```

2. In `app/models/blog.rb` `generate_slug` method, change:
   ```ruby
   self.slug ||= title.parameterize if title.present?
   ```
   to:
   ```ruby
   self.slug = title.parameterize if slug.blank? && title.present?
   ```

**Verify:**
- `grep -q 'if blog.slug.present?' app/views/admin/blogs/index.html.erb`
- `grep -q 'slug.blank?' app/models/blog.rb`
- `bundle exec rspec spec/requests/admin/blogs_spec.rb spec/models/blog_spec.rb`

**Done:** View link shows for any blog with a slug; slug auto-generates when form submits blank.
