## Component Inventory

### 1. Keywords — Tag-Chip Input

**Stimulus controller:** `keywords-input` (new)

**Layout:** Full-width (`md:col-span-2`) within existing `<div class="grid grid-cols-1 md:grid-cols-2 gap-6">`.

**Visual anatomy:**
```
[ Label: "Keywords" ]
┌──────────────────────────────────────────────────────┐
│ [chip: seo ×] [chip: marketing ×] [chip: b2b ×]     │
│ ┌────────────────────────────────────────────────┐   │
│ │ Type keyword and press Enter...                │   │
│ └────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────┘
[ Hint: Press Enter to add a keyword. Click × to remove. ]
<input type="hidden" name="blog[keywords]" value="[\"seo\",\"marketing\",\"b2b\"]">
```

**Chip appearance:**
- Class: `inline-flex items-center gap-1 px-2 py-1 rounded-full text-xs font-semibold bg-pink-50 text-pink-700 border border-pink-200`
- Remove button inside chip: `×` character, `h-8 w-8 inline-flex items-center justify-center rounded-full hover:bg-pink-200 transition-colors`, `aria-label="Remove {keyword}"`

**Text input (within chip container):**
- Class: `flex-1 min-w-[120px] px-2 py-1 text-sm outline-none bg-transparent`
- Wrapper div class: `flex flex-wrap items-center gap-2 w-full px-3 py-2 border border-gray-300 rounded-md focus-within:ring-2 focus-within:ring-pink-500 focus-within:border-transparent min-h-[42px]`

**Keyboard behaviour:**
- `Enter` key: creates chip from current input text, clears input, prevents form submit
- `Backspace` on empty input: removes last chip
- Comma: treated same as Enter (creates chip)

**Hidden field strategy:**
- One `<input type="hidden" name="blog[keywords][]">` per chip, value = chip text
- Stimulus syncs hidden fields on every add/remove action
- Rails strong params permit `keywords: []` to receive array

**States:**
- Default: border `border-gray-300`, focus-within ring `ring-pink-500`
- Empty (no chips): show placeholder text in input "Type keyword and press Enter..."
- Filled: chips rendered left-to-right, input appended after last chip

### 4. FAQ Builder — Collapsible Section

**Stimulus controller:** `faq-builder` (new)

**Position:** Below the metadata grid and above the excerpt field, as a full-width collapsible block.

**Collapsible mechanism:** Native HTML `<details>`/`<summary>` element — no Stimulus needed for collapse toggle. `<details>` open by default only when `@blog.faq_schema.present?`; otherwise closed.

**Visual anatomy — collapsed state (no FAQs):**
```
┌─ ▶ FAQ Schema (0 pairs) ────────────────────────────────────┐
└─────────────────────────────────────────────────────────────┘
```

**Visual anatomy — expanded state:**
```
┌─ ▼ FAQ Schema ──────────────────────────────────────────────┐
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ Q: [ text input — Question                         ] │  │
│  │ A: [ textarea 2 rows — Answer                      ] │  │
│  │                                        [Remove row] │  │
│  └──────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ Q: [ text input                                    ] │  │
│  │ A: [ textarea 2 rows                               ] │  │
│  │                                        [Remove row] │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  [ + Add FAQ Pair ]                                         │
│                                                             │
│  Hint: FAQ pairs are rendered as FAQPage JSON-LD in the     │
│  page head. Leave empty to omit FAQ schema entirely.        │
└─────────────────────────────────────────────────────────────┘
```

**`<details>` wrapper classes:**
- `<details>` element: `border border-gray-200 rounded-md bg-gray-50`
- `<summary>` element: `flex items-center justify-between px-4 py-3 cursor-pointer select-none text-base font-semibold text-gray-700 hover:bg-gray-100 rounded-md transition-colors list-none`
- Inner content wrapper: `px-4 pb-4 space-y-4`

**FAQ row card:**
- Class: `bg-white border border-gray-200 rounded-md p-4 space-y-3`
- Data attribute: `data-faq-builder-target="row"`

**Row field classes:**
- Question label: `block text-sm font-semibold text-gray-700 mb-1`
- Question input: `w-full px-3 py-2 border border-gray-300 rounded-md text-sm focus:ring-2 focus:ring-pink-500 focus:border-transparent`
  - `name="blog[faq_schema][][question]"`, `data-faq-builder-target="question"`
- Answer label: `block text-sm font-semibold text-gray-700 mb-1`
- Answer textarea: `w-full px-3 py-2 border border-gray-300 rounded-md text-sm focus:ring-2 focus:ring-pink-500 focus:border-transparent`, `rows="2"`
  - `name="blog[faq_schema][][answer]"`, `data-faq-builder-target="answer"`

**Remove button:**
- Class: `inline-flex items-center gap-1 px-3 py-1 text-xs font-semibold text-red-600 border border-red-200 rounded-md hover:bg-red-50 transition-colors`
- Label: "Remove"
- `data-action="click->faq-builder#removeRow"`
- Confirmation: none — FAQ row removal is low-stakes and reversible before form submit

**Add button:**
- Class: `inline-flex items-center gap-2 px-4 py-2 text-sm font-semibold text-gray-700 border border-gray-300 rounded-md hover:bg-gray-50 transition-colors`
- Label: "+ Add FAQ Pair"
- `data-action="click->faq-builder#addRow"`

**Hidden field / param strategy:**
- Each row serializes as `blog[faq_schema][][question]` and `blog[faq_schema][][answer]`
- Rails strong params: `faq_schema: [[:question, :answer]]`
- Model serializes array of hashes → JSON stored in `faq_schema` text column
- Empty pairs (both fields blank) are stripped server-side before save

## Interaction Contracts

### Keywords Controller (`keywords-input`)

| Trigger | Action | Result |
|---------|--------|--------|
| `keydown Enter` on text input | `addChip` | Chip created from input value; input cleared; hidden fields synced |
| `keydown ,` on text input | `addChip` | Same as Enter; comma consumed, not inserted |
| `keydown Backspace` on empty input | `removeLastChip` | Last chip removed; hidden fields synced |
| `click` on chip `×` button | `removeChip(event)` | That chip removed; hidden fields synced |
| Form submit | (no extra action) | Hidden fields already in sync; Rails receives `blog[keywords][]` array |

Edge cases:
- Blank/whitespace-only input on Enter: no chip created, input cleared silently
- Duplicate keyword: chip created (no de-dup — server normalizes if needed)
- Value trimming: trim whitespace before chip creation

### FAQ Builder Controller (`faq-builder`)

| Trigger | Action | Result |
|---------|--------|--------|
| `click` on "+ Add FAQ Pair" | `addRow` | New row HTML appended to row container; inputs focused on question field |
| `click` on "Remove" button | `removeRow(event)` | Closest row element removed from DOM |
| `<details>` open | (native browser) | Rows revealed; no Stimulus action needed |
| `<details>` close | (native browser) | Rows hidden; data preserved in DOM |

New row template: cloned from a `<template>` element in the DOM with empty question/answer inputs. Indices in `name` attributes use array notation `blog[faq_schema][][question]` — Rails handles ordering without explicit indices.

## Accessibility Notes

- Chip remove buttons: `aria-label="Remove {keyword}"` on each `×` button
- Keywords input wrapper: `role="group"`, `aria-label="Keywords"` on outer div
- FAQ rows: each row is a `<fieldset>` with `<legend class="sr-only">FAQ pair {n}</legend>` for screen reader grouping
- Collapsible `<details>`/`<summary>`: native browser accessibility — no ARIA role override needed
- All new inputs follow existing focus ring pattern (`focus:ring-2 focus:ring-pink-500`)
- File inputs: existing `accept="image/*"` pattern preserved for OG image
