## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-17 | Spoofing | Admin::BlogsController#update (faq_schema param) | mitigate | Admin::BaseController authentication+authorization enforced |
| T-03-18 | Tampering / Stored XSS via `</script>` injection in JSON-LD | render_faq_schema output | mitigate | `json_escape(schema.to_json)` in `content_tag :script` — same pattern as the existing three JSON-LD helpers; Test 4 explicitly verifies `</script><script>alert(1)</script>` payload is escaped (also fixes SEC-02 inconsistency on render_article_schema line 84) |
| T-03-19 | Tampering / Stored XSS in visible FAQ HTML | `_faq_section.html.erb` `<dt><%= pair['question'] %></dt>` and `<dd><%= pair['answer'] %></dd>` | mitigate | ERB auto-escapes by default; no `raw`, `html_safe`, or `sanitize` on FAQ text; HTML entities encoded; markup-only payloads render as escaped text not executable code |
| T-03-20 | Tampering / Strong params injection | `faq_schema: [:question, :answer]` permit | mitigate | Strong params permit only those two keys per hash element; extra keys (e.g. `script_url`, `xss_payload`) silently filtered |
