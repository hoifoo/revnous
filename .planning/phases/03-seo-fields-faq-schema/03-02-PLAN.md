## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | Admin::BlogsController#update (canonical_url_override) | mitigate | Admin::BaseController authentication+authorization already enforced |
| T-03-08 | Tampering / Open redirect | `javascript:`, `data:`, `file:` schemes in canonical_url_override | mitigate | `URI::DEFAULT_PARSER.make_regexp(%w[http https])` model validation rejects all non-http(s) schemes; tested explicitly with `javascript:alert(1)` in Task 1 Test 5 |
| T-03-09 | Tampering / XSS | URL rendered into `href` attribute | mitigate | (a) Validation guarantees scheme is http/https before save; (b) ERB `<%= canonical_url %>` auto-escapes attribute value; no `raw` or `html_safe` |
| T-03-10 | Information Disclosure | Canonical URL visible in public HTML | accept | Canonical URLs are intentionally public SEO metadata |
| T-03-11 | Tampering | Malformed URL bypassing regex | mitigate | URI::DEFAULT_PARSER.make_regexp is the same regex used in production for User#linkedin_url since Phase 2; proven to reject `javascript:` schemes |
</threat_model>
