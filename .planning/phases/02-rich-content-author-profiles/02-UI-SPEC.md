## Spacing Scale

Identical to Phase 1 — Tailwind CSS 4 default 4px base scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps within toolbar button groups; BubbleMenu button gaps |
| sm | 8px | Button internal padding (compact toolbar + BubbleMenu buttons) |
| md | 16px | Default element spacing; form field gaps; author card internal padding |
| lg | 24px | Section padding within form card; author card outer padding |
| xl | 32px | Form card internal padding (`p-8` = 32px) |
| 2xl | 48px | Major section separation |
| 3xl | 64px | Admin navbar height (`h-16` = 64px); sticky toolbar `top` offset |

**Exceptions for Phase 2:**
- Avatar image size in author card (public show page): 64px × 64px (`w-16 h-16`) — fixed, not on scale
- Avatar image size in admin user form preview: 80px × 80px (`w-20 h-20`) — fixed, not on scale
- BubbleMenu button size: `h-8 w-8` (32px) — smaller than standard `h-9 w-9` toolbar buttons because the BubbleMenu floats over content and must not obstruct reading
- Image upload drag zone minimum height: 120px (`min-h-[120px]`) — below the 4px scale but a standard touch affordance
- Resize handle touch target: 12px × 12px corner handles — visual only, not on scale (the grab area is larger than the visual dot)

## Typography

All from Phase 1 — no new type roles introduced in Phase 2.

**Weights (exactly 2):** `font-normal` (400) for body text and `font-semibold` (600) for all labels, headings, and button text.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 14px (`text-sm`) | 600 (`font-semibold`) | 1.25 | Form field labels; author card "Author" section heading |
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 | Helper text; author bio text; spacing dropdown helper |
| Heading | 30px (`text-3xl`) | 600 (`font-semibold`) | 1.2 | Page headings ("Edit User", "Users", "New User") |
| Small/meta | 12px (`text-xs`) | 400 (`font-normal`) | 1.4 | Author job title in the author card; table header labels in admin index |

**Author card public page typography (inherits show page styles):**
- Author name: `text-base font-semibold text-gray-900` (16px / 600)
- Author job title: `text-sm text-gray-500` (14px / 400)

- Social link text: `text-sm text-pink-600 hover:text-pink-700` (14px / 400) — matches existing site link color

**WYSIWYG table typography inside `.ProseMirror`:** Controlled entirely by `prose prose-lg` (Tailwind Typography). Do not declare custom table cell font sizes — they must inherit from the prose context automatically.

Inserted into the Tiptap document as a custom node at the cursor position while upload is in progress. Replaced by `<img>` on success, removed on error.

Visual appearance in the editor:

`animate-spin` is a Tailwind utility — no custom CSS needed. The spinner uses CSS animation (no JS animation loop).

### 5. Alt Text Prompt

Triggered immediately after upload completes, before injecting `<img>` into the editor. Uses a native `window.prompt()` dialog — consistent with Phase 1's link URL prompt pattern:

```javascript
const alt = window.prompt('Enter alt text for this image (required):')
if (alt === null) {
  // User cancelled — remove placeholder, abort injection
} else {
  // Inject <img src="..." alt="..."> into editor
}
```

Prompt label: `'Enter alt text for this image (required):'`

If the user dismisses (Cancel): remove the upload placeholder, do not inject. If the user submits an empty string: inject with `alt=""` (decorative image — acceptable for admin tool).

### 6. Inline Image Resize Handles

The Tiptap Image extension is configured with `@tiptap/extension-image` (custom `resizable` option or community wrapper). Resize handles appear at the four corners of a selected image.

Visual spec:
- Handle: 8px × 8px filled square, `bg-pink-600`, `border border-white`, `rounded-sm`
- Position: absolutely positioned at each corner of the image node
- Cursor: `cursor-nwse-resize` (NW/SE corners), `cursor-nesw-resize` (NE/SW corners)
- Image selected state: `ring-2 ring-pink-500` on the `<img>` element

Width storage: `width` attribute on `<img>` (e.g. `width="480"`). No inline `style` — sanitizer allows `width` attribute.

On the published show page: images respect the `width` attribute but are capped by `max-width: 100%` via the prose container. No additional CSS needed — Tailwind Typography applies `img { max-width: 100% }` by default.

## Interaction Contracts

### Table Insert

- Clicking the (now-active) table toolbar button calls `editor.chain().focus().insertTable({ rows: 3, cols: 3, withHeaderRow: true }).run()`
- A 3×3 table with a header row is inserted at cursor position
- Immediately after insert, the BubbleMenu appears (cursor is now inside a table cell)
- The active state (`bg-pink-50 text-pink-700`) is applied to the table toolbar button while the cursor remains anywhere inside the table

### BubbleMenu Visibility

- Appears: when `editor.isActive('table')` — i.e. cursor is inside any table cell (td or th)
- Disappears: when cursor moves outside the table
- `shouldShow` callback: `({ editor }) => editor.isActive('table')`
- Positioning: `placement: 'top'`, offset 8px from the top of the selection

### Image Upload — Click Trigger

1. Admin clicks image toolbar button
2. Stimulus controller calls `this.imageFileInputTarget.click()`
3. Native file picker opens filtered to `image/*`
4. On file selection: `change` event fires on the hidden input
5. Upload proceeds (see Upload Flow below)

### Image Upload — Drag-and-Drop Trigger

1. Admin drags an image file over the editor container (`data-tiptap-editor-target="editor"`)
2. `dragover` event: add `border-pink-400 bg-pink-50` to the editor container; prevent default to allow drop
3. `dragleave` / `drop` event: remove the drag-over classes
4. On `drop`: extract `event.dataTransfer.files[0]`; validate it is an image type
5. If not an image: show flash-style error (`text-red-600 text-sm`) — "Only image files can be dropped here."
6. Upload proceeds (see Upload Flow below)

### Image Upload Flow

1. Insert placeholder node at cursor (spinner + "Uploading image..." text)
2. POST to `/rails/active_storage/direct_uploads` using the DirectUpload API (`@rails/activestorage`)
3. On upload progress: no progress bar — spinner is sufficient for admin tool
4. On success: show `window.prompt('Enter alt text for this image (required):')` dialog
   - If cancelled: remove placeholder; abort injection
   - If submitted (including empty string): replace placeholder with `<img src="[rails_blob_path]" alt="[value]" width="[natural image width capped at 720]">`
5. On upload error: remove placeholder; show inline error message at the cursor position: "Image upload failed. Please try again." (`text-sm text-red-600`)

**Initial width:** On insert, set `width` attribute to the image's natural width capped at 720px (the prose column max content width). This prevents oversized images on first insert.

### Image Resize

1. Admin clicks an inserted image — Tiptap marks it as selected (`ring-2 ring-pink-500`)
2. Four corner handles appear (8×8px `bg-pink-600` squares)
3. Admin drags a corner handle: the `width` attribute updates in real time as the drag proceeds
4. On drag release: the new `width` value is committed to the Tiptap document
5. Handles are removed when the image is deselected (click elsewhere)

### Paragraph Spacing

- Admin changes the "Paragraph Spacing" `<select>` in the blog form
- Form submits normally (no JS); `spacing` value saved to `blogs.spacing` column
- On show page: `prose-paragraph-relaxed` class is added to `.prose` container if `spacing == 'relaxed'`
- No live preview in the editor — change is visible on the published post after save

### Author Dropdown

- Admin selects a user from the `author_id` dropdown
- Both `author` (text) and `author_id` (FK) are submitted; model/controller handles both
- If `author_id` is set, the published post shows the full author card; the plain-text `author` field is ignored on display
- Dropdown uses `include_blank: "— No author profile —"` — selecting blank clears `author_id`

### Admin User Destroy

- Confirmed with `data: { turbo_confirm: "Delete this user? Their blog posts will be unassigned but not deleted." }`
- On confirm: `DELETE /admin/users/:id`
- Controller: `dependent: :nullify` — sets `blogs.author_id` to NULL for all blog posts authored by this user
- Flash on success: "User was successfully deleted."
- Redirect to: `admin_users_path`

### Flash Messages (Admin Users)

All use existing admin flash pattern (no new component needed — pattern inherited from base layout):
- Create success: "User was successfully created."
- Update success: "User profile was successfully updated."
- Destroy success: "User was successfully deleted."
- Save error: "User could not be saved. Please review the errors below."

## Accessibility Contract

**Admin forms (new/edit user):**
- All inputs: `id` matching `for` on label (Rails `form.label` + `form.text_field` handle this automatically)
- All inputs: descriptive `aria-describedby` pointing to helper text `<p>` when helper is present
- Password fields: `autocomplete="new-password"` on create form; `autocomplete="current-password"` on email, `autocomplete="new-password"` on password fields in edit
- File input (avatar): `aria-label="Upload avatar image"` in addition to visual label

**BubbleMenu:**
- Container: `role="toolbar"` `aria-label="Table controls"`
- Each button: `type="button"` (prevent form submit) + `title` + `aria-label` (both set, same text)
- Buttons are keyboard reachable: tab enters the BubbleMenu when it is visible; Escape closes BubbleMenu and returns focus to editor

**Author card (public show page):**
- Avatar `<img>`: `alt="[user full_name]"` always set
- Initials placeholder `<div>`: `aria-label="[user full_name]"` on the container; initials span has `aria-hidden="true"`
- Social link `<a>` elements: `rel="noopener noreferrer"` on all `target="_blank"` links
- "Written by" label and name are in reading order (label precedes name in DOM)

**Image in editor:**
- Alt text is required at upload time (prompt enforces it, empty string is acceptable)
- Inserted `<img>` always has `alt` attribute (never missing)

**Drag-and-drop:**
- Drop zone does not require drag — click trigger is always available as an alternative
- Drop zone error messages are inserted as visible text (not ARIA-only)

**Admin users index table:**
- `<table>` with `<thead>` and `<th scope="col">` on each header
- Avatar in table: `alt="[user full_name]"` always set; initials placeholder: `aria-label="[user full_name]"` on div

## Security Contract (Phase 2 additions to SEC-01)

**Sanitizer ALLOWED_TAGS additions** (extends Phase 1 list in `Blog::ALLOWED_TAGS`):
- Add: `table`, `thead`, `tbody`, `tfoot`, `tr`, `th`, `td`, `colgroup`, `col`, `figure`, `figcaption`
- (Phase 1 already allowed `img`)

**Sanitizer ALLOWED_ATTRIBUTES additions** (extends Phase 1 list in `Blog::ALLOWED_ATTRIBUTES`):
- Add: `width` (for image resize), `colspan`, `rowspan`, `scope` (for table accessibility)
- (Phase 1 already allowed `src`, `alt`, `height` on `img`; `href`, `target`, `rel` on `a`)

**Avatar image:** Served via ActiveStorage — `rails_blob_path(user.avatar)`. Not an external URL. No additional sanitization needed.

**LinkedIn / Twitter URLs on author card:** Rendered as `link_to url, target: "_blank", rel: "noopener noreferrer"`. The URL values must be validated in the User model (`validates :linkedin_url, format: { with: URI::DEFAULT_PARSER.make_regexp(%w[http https]), allow_blank: true }`). Do not use `raw` on user-supplied URL values.
