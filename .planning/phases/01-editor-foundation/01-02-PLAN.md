<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied attributes → JSON serialization → HTML output | Strings entered by admins (title, description, excerpt) flow through `.to_json` and become the body of `<script type="application/ld+json">` blocks rendered on public pages |
| Toolbar HTML attributes → Stimulus action dispatch | The `data-action` and `data-tiptap-editor-level-param` strings on toolbar buttons drive controller method dispatch — but they are author-controlled (Rails view template), not user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | `app/helpers/application_helper.rb` JSON-LD script blocks | mitigate | All four `content_tag :script, ..., type: "application/ld+json"` calls wrap their payload in `json_escape(schema.to_json)`. `json_escape` from `ERB::Util` escapes `<`, `>`, `&`, U+2028, U+2029 as `<`-style sequences which are valid inside a JSON string. Closes the pre-existing SEC-02 vulnerability where `</script>` inside any field (title, description, excerpt, breadcrumb name, product name) could terminate the script tag early and inject arbitrary script. |
| T-01-08 | Tampering | `setLink` — user pastes `javascript:` URL into the prompt | mitigate | `editor.chain().setLink({ href: url })` defers to Tiptap's Link extension URL filter (which rejects `javascript:` and `data:` schemes). Even if a malicious URL passes the editor, `sanitize_body` in T-01-01 (Plan 01) strips disallowed `href` values via `Rails::Html::SafeListSanitizer` URL-scheme filtering. Defense in depth: client-side editor + server-side sanitizer. |
| T-01-09 | Information Disclosure | Inline SVG icons | accept | SVGs are static markup rendered from the ERB template — no user input flows into them. No risk. |
| T-01-10 | Denial of Service | Long `window.prompt` URL or pathological undo history | accept | `setLink` rejects empty cleanly; the prompt itself is browser-native and cannot DoS the page. Tiptap's undo/redo history is bounded by StarterKit defaults (~100 ops). Acceptable for an admin-only tool. |
